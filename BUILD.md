# متطلبات بناء تطبيق الكراج

قبل `npm run build` تأكّد من التالي على جهاز البناء:

## 1) الوصول إلى نطاق cdn.sheetjs.com (مهم)

اعتمادية تصدير Excel (`xlsx`) مثبَّتة من الموزّع الرسمي لـ SheetJS وليست من npm:

```json
"xlsx": "https://cdn.sheetjs.com/xlsx-0.20.3/xlsx-0.20.3.tgz"
```

السبب: نسخة npm عالقة على `0.18.5` وفيها ثغرة معروفة (CVE-2023-30533)، والإصدارات
المُصلَّحة (0.20.x) تُوزَّع من `cdn.sheetjs.com` فقط. لذلك أي `npm install` نظيف
يحتاج **وصولاً شبكياً إلى `cdn.sheetjs.com`**. لو كان جهاز البناء خلف بروكسي أو
جدار ناري يحجب هذا النطاق، سيفشل التثبيت برسالة غير واضحة — اسمح بالنطاق أولاً.

## 2) سرّ الترخيص (GARAGE_LICENSE_SECRET)

سرّ قفل الجهاز (M4 من تقرير المراجعة) يُحقَن وقت البناء ولا يعيش في الكود المصدري.
مصدره بالأولوية:

1. متغيّر البيئة `GARAGE_LICENSE_SECRET`، أو
2. ملف `electron/.license-secret` (مُستبعَد من git).

```bash
# إمّا عبر متغيّر بيئة:
GARAGE_LICENSE_SECRET="<السر>" npm run build

# أو ضع السر في الملف (مرّة واحدة على جهاز البناء):
echo -n "<السر>" > electron/.license-secret
npm run build
```

**تحذير:** كل الأجهزة يجب أن تبني بنفس السرّ. سرّ مختلف = رفض قواعد بيانات كل
الأجهزة المرخّصة حالياً. احتفظ بنسخة آمنة من السرّ خارج جهاز التطوير. بناء الإنتاج
بلا سرّ يطبع تحذيراً صريحاً ويستخدم سرّاً بديلاً غير آمن (للتطوير فقط).

## 3) تصفير آثار الترخيص على أجهزة التطوير (reset-dev-license)

أثناء التطوير والاختبار المتكرر، كل تشغيل لنسخة محزَّمة يترك أثر ترخيص على الجهاز
(مفتاح ريجستري `HKCU\Software\GreenLineGarage` + ملف مخفي `.glg-lic` + صفوف في
`garage.db`). لو بُنيت نسخة جديدة بسرّ مختلف (أو تغيّر أي شيء في التوقيع)، ستظهر
رسالة **"غير مصرح"** عند تشغيلها على نفس الجهاز — هذا سلوك الحماية H4 الطبيعي
وليس خطأً في الكود.

لتصفير الجهاز قبل تجربة نسخة جديدة:

```bash
npm run reset-dev-license              # مع سؤال تأكيد Y/N
npm run reset-dev-license -- -Force    # بدون تأكيد
```

السكربت (`scripts/reset-dev-license.ps1`) يحذف مفتاح الريجستري والملف المخفي،
و**ينقل** (لا يحذف) مجلد بيانات البرنامج `%APPDATA%\car-repair-shop` (هذا هو
userData الفعلي للنسخة المحزَّمة ولوضع التطوير معاً، لأن `productName` غير
معرَّف في package.json) إلى نسخة احتياطية بلاحقة `_OLD_BACKUP_<طابع زمني>`،
فيمكن استرجاع أي بيانات لاحقاً بإعادة تسمية المجلد. (يشمل السكربت أيضاً
`%APPDATA%\GreenLineGarage` احتياطاً لو تغيّر الاسم مستقبلاً.)

> ⚠️ **تحذير:** هذا السكربت لأجهزة التطوير والاختبار **فقط**. لا يُشغَّل إطلاقاً
> على جهاز عميل حقيقي — تشغيله هناك يفكّ ربط الترخيص ويُخفي بيانات العميل في
> مجلد احتياطي، وأي تشغيل تالٍ سيبدأ بقاعدة بيانات فارغة.

## 4) ملاحظة ويندوز — صلاحية الروابط الرمزية

`electron-builder` يفكّ أدوات التوقيع التي تحوي symlinks. فعّل **"وضع المطوّر"**
في إعدادات ويندوز (Settings → Privacy & security → For developers) لتفادي خطأ
`Cannot create symbolic link: A required privilege is not held by the client`.
